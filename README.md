# telemost-recorder

Сервис на Python для автоматического входа во встречи Яндекс.Телемоста через `chromium-browser` и аудиозаписи встречи по ежедневному расписанию из конфигурации.

## vibecoding notice

Проект полностью завайбкожен и предоставляется "как есть", без гарантии работоспособности

## Что делает

Два режима: 
1. запускается как долгоживущий процесс и каждый день проверяет расписание из `TELEMOST_SCHEDULE`, в заданные часы заходит на созвон
2. запускается в режиме "записать один созвон прямо сейчас"

Дальше в обоих режимах:
- открывает указанную ссылку Телемоста в отдельном окне Chromium
- нажимает до двух баннеров `Понятно`, если они появляются
- меняет имя гостя на `Телемостер` (конфигурируется в .env)
- выключает микрофон и камеру перед подключением
- нажимает `Подключиться`
- пишет только звук Chromium через отдельный временный PulseAudio/PipeWire sink в аудиофайл
- останавливает запись после N секунд тишины

## Типовой сценарий

1. В `.env` задается ссылка на встречу, имя и ежедневное расписание.
2. Сервис запускается один раз и остается работать в фоне.
3. Каждый день в указанные моменты времени он сам подключается к встрече и начинает запись.
4. Когда во встрече нет звука N секунд подряд, запись останавливается, а браузер закрывается.

`TELEMOST_SCHEDULE` задается списком времени в формате `HH:MM[,HH:MM]*`.

Примеры:

- `TELEMOST_SCHEDULE=09:00` — один запуск каждый день в 09:00
- `TELEMOST_SCHEDULE=09:00,14:30,18:45` — несколько запусков каждый день

## Требования

- Linux с активной графической сессией и PulseAudio
- `chromium-browser` (в файрфоксе телемост у меня не работает)
- `ffmpeg`

## Установка

```bash
cp .env.example .env
uv sync
```

После копирования `.env.example` нужно подправить как минимум:

- `TELEMOST_URL` — ссылка на нужную встречу
- `TELEMOST_SCHEDULE` — ежедневное расписание


Проект сам создает отдельный virtual sink и записывает его monitor source.

## Команды

Одноразовый немедленный запуск записи:

```bash
uv run telemost-recorder once --start-now

# или ./start-once-now.sh
```

Во время `once --start-now` можно остановить процесс через `Ctrl+C`. В этом случае текущая запись завершается корректно: `ffmpeg` штатно закрывает файл, после чего процесс выходит.

Долгоживущий сервис с использованием расписания:

```bash
uv run telemost-recorder run

# или ./start-monitoring.sh
```

## Конфигурация

Через `.env` или переменные окружения:

- `TELEMOST_URL`
- `TELEMOST_DISPLAY_NAME`
- `TELEMOST_SCHEDULE` — ежедневное расписание в формате `HH:MM,HH:MM`
- `TELEMOST_RECORDINGS_DIR`
- `TELEMOST_AUDIO_BACKEND` — только `pulse`
- `TELEMOST_AUDIO_SINK_NAME` — префикс для временного sink, в который будет направлен Chromium
- `TELEMOST_WINDOW_SIZE` — размер окна Chromium для стабильной автоматизации
- `TELEMOST_WINDOW_POSITION` — положение окна Chromium
- `TELEMOST_SILENCE_TIMEOUT_SECONDS`
- `TELEMOST_SILENCE_MIN_DETECT_SECONDS`
- `TELEMOST_SILENCE_NOISE_DB`
- `TELEMOST_CHROMIUM_PATH`
- `TELEMOST_CHROMIUM_PROFILE_DIR`
- `TELEMOST_BROWSER_LAUNCH_TIMEOUT_SECONDS`
- `TELEMOST_JOIN_TIMEOUT_SECONDS`
- `TELEMOST_POST_JOIN_DELAY_SECONDS`
- `TELEMOST_FFMPEG_LOGLEVEL` — `error`, `warning` или `info`; влияет на то, какие сообщения `ffmpeg` попадут в логи сервиса

## Замечания

- Файл записи аудио-only: проект не пытается захватывать видео окна встречи.
- В запись попадает только звук Chromium, потому что для него поднимается отдельный временный sink; остальные системные звуки в файл не должны попадать.
- Перед входом бот явно выключает микрофон и камеру на prejoin-экране, затем уже нажимает `Подключиться`.
- Если аудиозахват не работает, команда `check` завершится с ошибкой и подскажет причину.
- При остановке через `Ctrl+C` во время активной записи сервис сначала корректно завершает текущий аудиофайл, а затем выходит.
- Расписание всегда интерпретируется по локальному системному времени компьютера.
